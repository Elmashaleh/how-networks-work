## The pitch

Every networking explainer on the internet does the same thing: throws OSI layers at you and hopes something sticks. This one doesn't.

This one hands you an envelope.

Because that's the trick — the internet isn't some incomprehensible alien machine. It's a **postal system that never sleeps**, running the same four rules you already know from mailing a birthday card: someone sends, someone receives, there's an address, and there's a way to know it arrived. Everything else — IP, MAC, routers, TCP — is just that idea, enforced with terrifying discipline, a few billion times a second.

So instead of a wall of text, you get a working post office. Type a message. Watch it get torn apart, stamped, routed through three offices that have never met, and rebuilt on the other side — correctly, every time, whether you sabotage a packet on purpose or not.

## What you can actually *do* here

This isn't a scroll-and-read page. It's a scroll-and-*mess-with-it* page.

| Section | What happens when you touch it |
|---|---|
| **The Post Office** | Send a real message. Watch it fragment, route, and reassemble — live, with a fully working simulated packet-loss + resend. |
| **Passport vs. mailing label** | Move to a "new network" and watch your IP change while your MAC doesn't even flinch. |
| **Build-your-own-route** | Pick the path yourself. Any combination of offices. It still arrives. That's the whole point. |
| **Flip-card packets** | Flip torn fragments in whatever order you want. They reassemble correctly anyway. |
| **The journey track** | A tiny envelope rides down the left edge of the page as you scroll — you *are* the packet, the whole way through. |
| **The wax seal** | Break it. You've earned a certificate. Put your name on it. |

No frameworks, no build step, no dependencies you have to install. It's one `index.html` file that does all of that by itself.

## Why it looks like a 1930s airmail envelope and not a terminal

Because "dark mode + neon cyan" has been done to death, and because the paper this site is built from is *about* the mail analogy — so the site should feel like the thing it's explaining, not like every other tech blog on the internet. Cream paper, postal red, foil gold, typewriter type, a wax seal that actually cracks. If you're going to teach someone how packets travel, you might as well make them feel like they're standing at the counter watching it happen.
